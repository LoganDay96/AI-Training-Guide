Installation

Prerequisites

  • A Walmart laptop (Windows or Mac)
  • Access to the internet (on VPN or in-office)
  • Windows PowerShell (Windows) or Terminal (Mac)

Install Steps

  1. Go to puppy.walmart.com
  2. Sign in with Walmart Auth
  3. Open PowerShell (Windows) or Terminal (Mac)
  4. Copy the install command from the website for your OS
  5. Paste and press Enter - installation takes 1-3 minutes

Running Code Puppy

Open PowerShell/Terminal and type:

code-puppy
Pro Tip Navigate to your project folder first with cd path\to\folder, then run code-puppy.

Choosing the Right Model

Important If Code Puppy seems "dumb," you're probably on Haiku (the lightweight model). Switch to a full model!

Type /model to see options. Recommended models:

ModelBest For
Claude Opus 4.5Best overall for most tasks
ChatGPT 5.2Very good, more methodical
Gemini 3.0Great for images and front-end design

If one model gets stuck:

  1. Try 3 times, then switch models
  2. If still stuck, start a new conversation

Best Practices

The Waterfall Approach

  1. Clean your data first - Fix formatting, add clear headers
  2. Let AI understand it - Have it ask questions until it gets it
  3. Create a plan together - Don't jump to building
  4. Execute - Feed the plan to Code Puppy
  5. Iterate - Review and refine

Prompting Tips

Instead of...Try...

The Interview Technique

For complex tasks, use this magic prompt:

Help me come up with a plan. DON'T WRITE ANY FILES.
Ask me clarifying questions until you are 95% confident.
Provide options labeled a, b, c for me to choose from.

Context Management

Context Limits When conversations get long (50-70% of context used), you'll notice generic answers and forgotten instructions. Time for a new chat!

Preserving Progress

Before ending a long session:

Create a context.md file summarizing what we built, current state, and next steps.

When starting fresh:

Read the context.md file to understand where we left off.

Quick Wins

  • Launcher files: Ask for a batch file to double-click and launch your tool
  • Cache data: Avoid re-uploading data every time
  • Screenshot errors: Paste them directly into chat

Saving Your Work (Version Control Basics)

Here's a truth every developer learns the hard way: things break. When AI "improves" something and suddenly nothing works, you'll be grateful for backups.

Golden Rule Before asking AI to make any significant change, save a copy of what's currently working.

Simple Backup Strategy: Dated Folders

The easiest approach that works great:

  1. Find your project folder (e.g., my-dashboard)
  2. Copy the entire folder
  3. Rename the copy with today's date: my-dashboard-2026-01-22-working
When to SaveFolder Name Example
Before a big AI changemy-dashboard-2026-01-22-before-chart-update
After something worksmy-dashboard-2026-01-22-login-working
End of work sessionmy-dashboard-2026-01-22-end-of-day

The "Save a Working Version" Mindset

Train yourself to think: "It works! Time to save."

  • ‚úÖ Login page displays correctly? Save it.
  • ‚úÖ Chart shows real data? Save it.
  • ‚úÖ Export button works? Save it.

When Everything Breaks: Recovery Plan

  1. Stop making changes ‚Äî Don't let AI try to "fix" it (often makes it worse)
  2. Find your last backup ‚Äî Look for your most recent dated folder
  3. Replace the broken files ‚Äî Copy working files from backup
  4. Test it works ‚Äî Run your dashboard to confirm
  5. Try again with smaller changes ‚Äî Ask AI for one small change at a time

Undo Strategies

SituationWhat To Do
AI just made a bad changePress Ctrl+Z immediately in your editor
You saved the bad changeRestore from your dated backup folder
Multiple files are brokenReplace entire project folder with backup
Not sure what changedCompare files side-by-side with your backup
