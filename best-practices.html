Best Practices

These tips will help you get the most out of Code Puppy and avoid common pitfalls.

The Waterfall Approach

For complex projects, follow this structured approach:

  1. Clean your data first
    If your data is in Excel or a similar format, add helper columns and clean up any messy formatting before feeding it to Code Puppy.
  2. Let the AI understand your data
    Have Code Puppy (or ChatGPT) look at the data source and ask you questions until it fully understands what it's looking at.
  3. Create a framework together
    Work with the AI to build a plan before any code is written. Have it interview you until you both agree on the approach.
  4. Feed the framework to Code Puppy
    Once you have a clear plan, give it to Code Puppy and let it execute.
  5. Let it work
    Trust the process! Review the output and iterate as needed.

Prompting Tips

Be Specific

Vague prompts lead to vague results.

Instead of... Try...
"Make me a chart" "Create a bar chart showing monthly sales by region, with blue bars and a white background"
"Analyze this file" "Analyze this Excel file and find the top 10 customers by total revenue"
"Fix this" "This code throws an error on line 15. Here's the error message: [paste error]"

Give Examples

If you have a specific look or behavior in mind, show it:

I want the dashboard to look similar to this screenshot: [paste image]
Or like this website: [provide URL]

Screenshot Errors

When something goes wrong:

  1. Take a screenshot of the error
  2. Paste it directly into the chat
  3. Code Puppy can read images and will understand the error

Context Management

This is one of the most important concepts for working effectively with AI.

Context Window Limits Current models have a limit of 200k - 272k tokens per conversation. When the context window fills up (50-70% full), you'll notice:
  • Generic, less helpful replies
  • The agent forgets earlier instructions
  • Repeated or contradictory responses

When to Start a New Chat

If the agent starts giving you generic answers or forgets what you were working on, it's time for a fresh conversation.

Preserving Context Across Chats

Before ending a long session:

Please create a context.md file that summarizes:
- What we're building
- The current state of the project
- Any specific style or constraints we agreed on
- What we should work on next

When you start a new chat:

Please read the context.md file in this project folder
to understand where we left off.

Model Switching

Different models have different strengths. If you're stuck:

  1. Try a different model - Claude and ChatGPT approach problems differently
  2. Give it 3 attempts - If one model can't fix a bug in 3 tries, switch
  3. Start fresh - Sometimes a new conversation helps more than switching models
Model Strengths
Claude Sonnet Excellent at coding, writes detailed documentation
ChatGPT Codex Great at understanding directions, solid coding
Gemini Better at visual/front-end work and image understanding

Working with Visuals

ASCII Mockups

AI agents aren't always great at designing things from scratch. If you have a specific layout in mind:

  1. Describe your design to ChatGPT
  2. Ask it to create an ASCII mockup
  3. Feed that mockup to Code Puppy as a reference
I want a dashboard with a header, sidebar on the left,
and a main content area. Please create an ASCII mockup
of this layout that I can give to my coding agent.

Copying Existing Designs

If you like how something looks:

  1. Screenshot it
  2. Have ChatGPT create an ASCII mockup from the screenshot
  3. Give that to Code Puppy to recreate

Quick Wins

Create a Launcher

For tools you build, ask Code Puppy to create a launcher file:

Create a batch file I can double-click to automatically
open this tool in my browser.

Cache Your Data

For data processing tools:

Add caching so I don't have to upload the data every time.
Make sure new uploads replace the cached data.

Label Your Data

Clean, well-labeled data saves enormous amounts of time. Before starting any analysis:

  • Add clear column headers
  • Remove unnecessary columns
  • Fix any formatting issues
Save Progress Frequently When using VS Code or similar tools with GitHub, commit your changes often. Create a new branch before trying anything experimental or "squirrely."
