Troubleshooting

Common issues and how to resolve them.

Installation Issues

"code-puppy" is not recognized

Cause: PowerShell/Terminal hasn't refreshed since installation.

Solution:

  1. Close PowerShell/Terminal completely
  2. Open a new window
  3. Try running code-puppy again

If it still doesn't work, try reinstalling from puppy.walmart.com.

Connection or download errors

Cause: Network or proxy issues.

Solution:

  • Make sure you're connected to VPN or on the Walmart network
  • Try again in a few minutes - sometimes there are temporary outages
  • If the problem persists, try reinstalling

Permission denied errors

Cause: PowerShell doesn't have the right permissions.

Solution (Windows):

  1. Right-click on PowerShell
  2. Select "Run as Administrator"
  3. Try the installation again

Runtime Issues

Code Puppy starts but behaves strangely

Cause: You might be using a less capable model.

Solution:

  1. Type /model and press Enter
  2. Switch to Claude Opus 4.5 or ChatGPT 5.2
  3. Try your request again
Check Your Model First If Code Puppy seems "dumb" or gives unhelpful responses, the most common cause is using a lightweight model like Haiku instead of a full model like Claude Opus.

Responses are getting worse over time

Cause: The context window is filling up.

Solution:

  1. Save your progress by asking Code Puppy to create a context.md file
  2. Start a new conversation
  3. Point Code Puppy to the context.md file to resume

See the Best Practices page for more on context management.

Code Puppy can't find my file

Cause: The file path might be incorrect, or the file is in use.

Solution:

  • Make sure you're using the full file path
  • Close any programs that have the file open (especially Excel)
  • Try copying the file to a simpler path like C:\Temp\myfile.xlsx
Getting the Right Path In Windows File Explorer, hold Shift, right-click the file, and select "Copy as path" to get the exact path.

"The file is being used by another process"

Cause: Another program (usually Excel or Word) has the file open.

Solution:

  1. Close the file in the other program
  2. Try again

Or, copy the file to a new location and use that path instead.

Model-Specific Issues

One model keeps failing, but another works

This is normal! Different models have different strengths and weaknesses.

Solution:

  1. If one model can't solve a problem in 3 attempts, switch to another
  2. Try: Claude Opus, ChatGPT Codex, or Gemini
  3. If none work, start a completely new conversation

Model is very slow

Cause: The model might be under heavy load, or your request is complex.

Solution:

  • Wait for the response - complex tasks take longer
  • Try a different model if speed is critical
  • Break your request into smaller pieces

Agent Issues

BigQuery or Confluence agents aren't working

Cause: Authentication or access issues.

Solution:

  • Make sure you have the proper permissions for BigQuery/Confluence
  • Try restarting Code Puppy
  • Check if you need to re-authenticate

Custom agent not behaving as expected

Cause: The agent configuration might need adjustment.

Solution:

  1. Ask Code Puppy to show you the agent's current configuration
  2. Describe what behavior you want changed
  3. Have Code Puppy update the agent configuration

Still Having Issues?

Getting Help
  • Check the Code Puppy Teams channel for known issues
  • Ask your question there - the community is helpful!
  • For critical issues, contact the Code Puppy team directly
  • You can always reinstall from puppy.walmart.com
